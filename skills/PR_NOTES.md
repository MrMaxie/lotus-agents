# SKILL: PR_NOTES

## OUTPUT FILE
.local/pr-notes/<id>.md

## RULES

- describe WHAT changed (user perspective)
- avoid:
  - class names
  - implementation details

- include:
  - feature changes
  - behavior updates
  - relevant refactors

## UPDATE MODE

IF file exists:
- merge changes
- produce final version (NOT changelog)

## TEMPLATE HANDLING

- preserve:
  - screenshots sections
  - video sections
  - unknown sections

- DO NOT fill if unsure

## STYLE

- concise
- business-oriented
