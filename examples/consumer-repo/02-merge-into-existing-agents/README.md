# Example 02 - Merge Into Existing `AGENTS.md`

Use this when the repo already has an `AGENTS.md` and you are willing to edit
it directly.

```text
repo/
  AGENTS.md
  .local/
    AGENTS.md
    context.md
    templates/
    issues/
    issues-notes/
    questions/
```

Typical merge targets:

- read order
- `.local/` conventions
- issue and review artifact naming
- docs rules for `docs/specs/` and `docs/meetings/`
