# Local Overrides Example

Use this file only for machine-local or temporary workflow rules that should
override the shared contract on one machine.

## Good Uses

- machine-specific tool paths
- local-only execution preferences
- temporary local constraints

## Avoid

- restating the shared runtime contract
- durable project rules that belong in committed docs
- human instructions that are only relevant to one run
- redefining repository truth that should live in committed artifacts
